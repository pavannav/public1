### Exercise 1.1: Understanding Rebasing Concepts
**Objective**: Grasp the fundamental concept of git rebasing and its purpose.

**Tasks**:
1. Read and analyze the session content about rebasing
2. Create a visual diagram showing how rebasing changes commit history
3. Document the key differences between rebasing and merging
4. Write a brief explanation of when rebasing creates a "linear" history

**Deliverable**: Hand-drawn or digital diagram showing before/after rebase scenarios with explanatory notes.

### Exercise 1.2: Rebasing vs Merging Comparison
**Objective**: Compare and contrast rebasing with merging operations.

**Tasks**:
1. List 3 scenarios where merging is preferred over rebasing
2. List 3 scenarios where rebasing is preferred over merging
3. Document the visual differences in git history for both approaches
4. Create a decision matrix for choosing between rebase and merge

**Deliverable**: Comparison table and decision matrix document.
