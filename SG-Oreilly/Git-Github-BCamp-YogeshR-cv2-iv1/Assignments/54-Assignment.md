### Exercise 2.2: Task Creation as Issues vs Draft Cards
**Objective**: Master the distinction between creating formal issues and draft cards for task management.

**Tasks**:
1. Create "About page" feature as a formal issue with proper description and repository assignment
2. Create "Deploy to production" as a draft card for idea capture without full issue creation
3. Create "Contact us page" as another draft card example
4. Document the decision criteria: when to use formal issues vs when to use draft cards

**Task Creation Methods**:
```
Formal Issues:
- Create blank issue with detailed description
- Assign to repository, team members
- Add labels (enhancement, bug, documentation)
- Link to milestones

Draft Cards:
- Quick idea capture without full issue creation
- Useful for brainstorming and initial planning
- Can be converted to issues later if needed
```

**Deliverable**: Task creation guide with issues vs draft cards decision framework and implementation examples.

### Exercise 3.3: Automation and Integration Strategy
**Objective**: Understand and plan GitHub Projects automation for reducing manual updates and maintaining synchronization.

**Tasks**:
1. Analyze automation triggers: new issues automatically appear in To Do
2. Configure automation rules: pull requests linked and marked as In Progress
3. Plan merge automation: merged pull requests automatically move cards to Done
4. Design automation strategies that "keep the project board in sync with actual development work"

**Automation Workflow**:
```
Issue Creation: New issue → Automatic To Do placement
PR Linking: Pull request creation → In Progress movement
Status Updates: PR marked in progress → Column movement
Merge Completion: PR merged → Automatic Done placement
```

**Deliverable**: Automation strategy document with trigger configuration and synchronization protocol design for minimizing manual project board maintenance.
