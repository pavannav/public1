This session covers theoretical concepts about Git branch merging including:

- Purpose of merging: combining changes from separate branches
- Fast-forward merge: Simple pointer movement when main branch hasn't changed
- Three-way merge: Used when both branches have diverged with new commits
- Merge commit: Special commit with two parents preserving history of both branches
- Merge conflicts: Occur when both branches edit the same part of a file
- Common workflow: Merging feature branches back into main/master

These concepts prepare for hands-on merging demonstrations in subsequent sessions.
