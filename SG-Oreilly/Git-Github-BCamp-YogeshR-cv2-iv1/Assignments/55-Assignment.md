### Exercise 1.1: Understanding GitHub Wiki Fundamentals
**Objective**: Grasp the core concepts of GitHub Wikis as dedicated documentation spaces beyond single README files.

**Tasks**:
1. Analyze the distinction between GitHub Projects (task organization) and GitHub Wikis (knowledge base documentation)
2. Document how wikis serve as "knowledge bases" with organized documentation vs monolithic README files
3. Understand wiki capabilities: detailed guides, tutorials, technical specifications
4. Recognize the key benefits: better project growth support, team collaboration, welcoming new contributors

**Deliverable**: GitHub Wiki concept document with comparisons to README files and documentation benefits analysis.

### Exercise 1.2: Wiki Navigation and Access
**Objective**: Navigate to GitHub Wiki interface and understand the wiki creation workflow.

**Tasks**:
1. Navigate to a repository's "Wiki" tab and locate the wiki interface
2. Identify the "Create first page" option for new wiki initialization
3. Document the wiki creation prompt and page creation interface
4. Understand that wikis provide "dedicated space inside a repository" for organized documentation

**Wiki Access Process**:
```
Repository Navigation: Wiki tab location and access
Initial State: "Create first page" prompt for new wikis
Interface: Page creation with name and content requirements
Purpose: Knowledge base establishment for project documentation
```

**Deliverable**: Wiki navigation guide with interface access and initial creation workflow documentation.

### Exercise 1.3: Markdown Formatting and Editor Capabilities
**Objective**: Master GitHub Wiki's markdown formatting capabilities for professional documentation.

**Tasks**:
1. Use markdown headings with `# Getting Started` syntax for document structure
2. Practice inserting horizontal lines, images, bold/italic text formatting
3. Include code snippets and links within wiki documentation
4. Document all formatting options available in the wiki editor interface

**Markdown Formatting Options**:
```
Headings: # Heading 1, ## Heading 2, ### Heading 3
Text Formatting: **bold**, *italic*, code snippets
Structure: Horizontal lines, lists, numbered items
Media: Images, embedded content
Links: Internal wiki links, external references
```

**Deliverable**: Markdown formatting guide with comprehensive editor capabilities and professional documentation techniques.

### Exercise 2.1: Home Page Creation and Basic Content
**Objective**: Create the foundational wiki home page with essential project information.

**Tasks**:
1. Create home page with descriptive title and welcome messaging
2. Add project overview content: "Welcome to my first repository Wiki"
3. Include resource description: "Here you will find notes and resources about this project"
4. Understand the role of home page as wiki entry point and project overview

**Home Page Content Structure**:
```
Title: Home (or project-specific name)
Welcome Message: Repository introduction and purpose
Resource Overview: What documentation and resources are available
Navigation: Entry point to other wiki pages and sections
```

**Deliverable**: Home page creation guide with essential content structure and project introduction best practices.

### Exercise 2.2: Preview and Publishing Workflow
**Objective**: Master the wiki page preview and publication process with commit messaging.

**Tasks**:
1. Use the "Preview" button to verify formatting before publication
2. Review the preview showing "same message that we had written in the description box"
3. Configure commit message (default: "Initial Home page") for version tracking
4. Execute save page operation and verify successful wiki page publication

**Publishing Workflow**:
```
Content Creation: Write markdown content in edit mode
Preview Verification: Click preview to see formatted result
Commit Documentation: Add descriptive commit message for changes
Publication: Save page to publish wiki content
```

**Deliverable**: Preview and publishing workflow guide with verification and version control integration.
