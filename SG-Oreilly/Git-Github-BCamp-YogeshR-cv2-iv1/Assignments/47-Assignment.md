### Exercise 1.1: Understanding Issue Creation Scenarios
**Objective**: Identify situations that warrant creating GitHub issues in open source projects.

**Tasks**:
1. Analyze the demonstration scenario: finding bugs or suggesting features in forked repositories
2. Document different types of issues: bugs, typos, feature requests, improvements
3. Understand that issues serve as project task lists or bug trackers
4. Identify the protocol: open issues before directly changing code in others' repositories

**Deliverable**: Issue creation scenarios document with examples of appropriate use cases.

---

### Exercise 1.2: Repository Navigation and Issues Tab Access
**Objective**: Navigate to project repositories and locate the Issues management interface.

**Tasks**:
1. Open the main project repository (not the fork) in a new browser tab
2. Locate the "Issues" tab at the top of the repository navigation
3. Verify that issues are enabled for the project
4. Analyze current issue status: zero open issues, zero closed issues
5. Read any contributing guidelines mentioned on the issues page

**Deliverable**: Repository navigation guide with Issues tab location and current status documentation.

---

### Exercise 1.3: Issue Creation with Proper Formatting
**Objective**: Create a well-structured GitHub issue with descriptive title and detailed explanation.

**Tasks**:
1. Click "New issue" button to start the issue creation process
2. Create a descriptive title: "Fixed typos in readme"
3. Write a detailed description explaining specific changes needed
4. Include concrete details: removing test content, adding CSS and JavaScript to technologies section
5. Preview the issue description before submission using GitHub's preview feature

**Commands** (Issue content example):
```
Title: Fixed typos in readme
Description:
- Remove test content from README
- Add CSS and JavaScript under "Technologies used" section
- Current section only mentions HTML but project includes CSS/JS
```

**Deliverable**: Successfully created and numbered GitHub issue with comprehensive description.

### Exercise 2.1: Issue Management and Organization Features
**Objective**: Explore GitHub's issue management and organizational capabilities.

**Tasks**:
1. Analyze the newly created issue details: issue number assignment, open status
2. Explore organizational features: assign people, add labels, link projects, set milestones
3. Understand the meaning of "open" status: problem hasn't been solved yet
4. Document how these features help organize larger projects

**Deliverable**: Issue management features documentation with organizational tool explanations.

---

### Exercise 2.2: Discussion Creation and Community Engagement
**Objective**: Create GitHub discussions as community forums for broader project conversations.

**Tasks**:
1. Navigate to the "Discussions" tab in the repository
2. Understand the difference: issues for concrete problems, discussions for community forums
3. Explore discussion categories: All discussions, Announcements, General, Ideas, Polls, Q&A
4. Create a new discussion in the Q&A category about project setup
5. Provide a clear title and description asking for setup guidance

**Deliverable**: Successfully created GitHub discussion with appropriate category selection.

---

### Exercise 2.3: Discussion vs Issue Differentiation
**Objective**: Understand when to use issues vs discussions for different types of project communication.

**Tasks**:
1. Compare the created issue (typo fixes) with the created discussion (setup questions)
2. Document the distinction: issues track bugs, tasks, specific improvements
3. Document the distinction: discussions handle open conversations, ideas, questions
4. Create guidelines for choosing between issues and discussions based on communication type

**Deliverable**: Decision matrix for choosing between GitHub issues and discussions.

---

### Exercise 3.1: Issue Commenting and Community Interaction
**Objective**: Practice engaging with GitHub issues through comments and community interaction.

**Tasks**:
1. Add constructive comments to existing issues using @username mentions
2. Practice professional issue commenting: "hey @username, it's a typo, we will look into it"
3. Understand how comments facilitate collaboration and problem-solving
4. Document the collaborative nature of issue resolution through community involvement

**Deliverable**: Issue commenting practice with documented community interaction examples.

---

### Exercise 3.2: Discussion Engagement and Knowledge Building
**Objective**: Engage with GitHub discussions to contribute to project knowledge bases.

**Tasks**:
1. Add replies to existing discussions sharing experiences or providing steps
2. Practice answering questions in the Q&A category
3. Understand how discussions create valuable knowledge bases over time
4. Mark appropriate answers when community members provide correct solutions

**Deliverable**: Discussion engagement documentation showing community knowledge building participation.

---

### Exercise 3.3: Project Collaboration Workflow Integration
**Objective**: Integrate issues and discussions into complete project collaboration workflows.

**Tasks**:
1. Document the complete workflow: identify problems → create issues → discuss solutions → implement fixes
2. Create a project communication strategy using both issues and discussions appropriately
3. Design protocols for maintainers to triage and respond to community input
4. Plan how open source projects can grow stronger communities through effective issue/discussion management

**Deliverable**: Complete project collaboration workflow document integrating issues and discussions effectively.
