## 09-Validating-Github-Copilot

### Overview

This module covers the steps required to select and access a GitHub Copilot service plan after installing the extension. It distinguishes between individual and business/enterprise plans, explains how organizational policies affect settings, and provides a walkthrough for verifying GitHub account status. The module also highlights how students and faculty can validate their educational status to receive GitHub Copilot and GitHub Codespaces hours for free as part of the GitHub Student Developer Pack.

### Key Concepts/Deep Dive

#### GitHub Copilot Service Plans

- GitHub Copilot requires a paid service subscription after installing the extension in your text editor.
- Users are directed to a landing page with several different service plans to explore before making a decision.
- Clicking "Get Started with Copilot" presents different access options depending on the user's context:
  - **Copilot Business or Enterprise**: Designed for organizations that need control over how developers use Copilot.
  - **Copilot for Individuals**: The recommended starting point for individual developers who want an AI assistant for personal coding projects.

#### Free Trial and Account Access

- A free trial is available when clicking "Get Started."
- Selecting the free trial redirects users to their GitHub settings under their personal account.
- If GitHub Copilot is already enabled (for example, provided by an organization), the user will see that access is granted through the organization and settings cannot be modified.

#### Organization-Enforced Policies

- When Copilot access is provided by an organization, certain features and policies are grayed out and cannot be changed by the individual user.
- These organizational restrictions are something to consider when evaluating the business-side plans.

#### Editor-Specific Installation Guidance

- After selecting Copilot, users must accept terms and confirm their intent to use GitHub Copilot.
- Documentation is available for installing the GitHub Copilot extension in supported editors, including:
  - Visual Studio Code
  - JetBrains
  - Visual Studio
  - Vim or NeoVim
- Users who need help finding these settings should scroll to the left panel and locate the Copilot section.

#### Validating GitHub Student Developer Status

- Educational users can access GitHub Copilot for free by validating their student or faculty status as part of the GitHub Student Developer Pack.
- Benefits include additional hours for GitHub Codespaces.
- The validation process is documented in a Microsoft Educator Developer Blog post that provides a step-by-step walkthrough.
- Users with an EDU email address or those affiliated with an educational institution (students, faculty, or staff) are eligible to apply.
- The validation process can vary in complexity depending on the specific institution but is often worthwhile for the free access provided.

### Lab Demos

- **Navigating to GitHub Copilot Settings**:
  1. Log in to your GitHub account.
  2. Scroll to the left panel and find the "Copilot" section.
  3. Click on "Copilot" to view service options and manage access settings.

- **Installing GitHub Copilot in Visual Studio Code**:
  1. From the Copilot settings page, select "Visual Studio Code."
  2. Follow the installation documentation provided to add the GitHub Copilot extension.

- **Validating Student/Developer Status**:
  1. Access the Microsoft Educator Developer Blog post linked in the Copilot settings.
  2. Follow the step-by-step instructions to authenticate and validate your student or faculty status.
  3. Upon successful validation, access GitHub Copilot and Codespaces hours through the GitHub Student Developer Pack.

### Expert Insight

**Real-world Application**: Choosing the correct Copilot plan ensures compliance with organizational policies and maximizes productivity. For students and educators, validating status provides free access and extended Codespaces hours, making it invaluable for academic projects.

**Expert Path**: Explore Business or Enterprise plans if managing multiple developers or enforcing AI coding policies. For individual use, start with the free trial or student validation process to evaluate Copilot before committing to a subscription.

**Common Pitfalls**: Attempting to modify settings when they are controlled by an organization; skipping the student validation process and missing out on free access.

**Lesser-Known Facts**: GitHub frequently updates its plans and access types, so checking the latest options on the official landing page is recommended before deciding.
