## Overview

This module guides you through creating a free GitHub account, which is essential for following along with this course. GitHub provides a platform for hosting Git repositories either by editing files directly on the web or by pushing changes from your local computer.

---

## Creating Your GitHub Account

### Accessing GitHub

1. Navigate to **github.com** in your web browser
2. GitHub accounts are completely free to create

### Choosing Your Username

> [!IMPORTANT]
> Your username becomes part of your GitHub Pages website URL and may be difficult or impossible to change later.

**Username Requirements:**
- Can only contain alphanumeric characters and hyphens
- No dots (periods) allowed
- Example: `CalebToleen` allows for the website URL `CalebToleen.github.io`

**Username Selection Tips:**
- Choose something professional and memorable
- Consider using it as your personal brand
- GitHub may suggest alternatives if your preferred username is taken

### Entering Account Details

1. **Username**: Enter your chosen username following the requirements above
2. **Email Address**: Use any valid email address (Gmail, Hotmail, Yahoo, etc.)
   - The email must not already be associated with an existing GitHub account
   - Invalid email addresses will be rejected
3. **Password**: Create a secure password for your account

### Account Verification

1. After signing up, you'll need to verify your email address
2. Check your email inbox for a verification message from GitHub
3. Click the verification link to activate your account
4. Once verified, you can sign in to GitHub

---

## Post-Account Creation Options

### Two Primary Workflows

Once your account is created and verified, you have two options for working with Git:

1. **Web-based editing**: Edit files directly on github.com using the web interface
2. **Local development**: Edit files on your local computer and push them to GitHub

> [!NOTE]
> Both workflows will be demonstrated in subsequent lessons of this course.

### Next Steps After Account Creation

1. Create your first repository
2. Install Git on your local machine (if planning to use local workflow)

### Expert Insight

**Real-world Application:**
GitHub accounts are essential for modern software development, enabling collaboration, version control, and portfolio showcasing. Your username often becomes your professional identity in the developer community.

**Expert Path:**
- Choose a username you'll be comfortable using professionally
- Consider setting up GitHub Pages early to create your developer portfolio
- Enable two-factor authentication for enhanced security
- Explore GitHub's free tier limitations to understand when you might need paid features

**Common Pitfalls:**
- Using temporary or throwaway email addresses that you might lose access to
- Selecting usernames with special characters that aren't allowed
- Not verifying email promptly, which blocks account activation
- Choosing usernames that might conflict with future professional branding

**Lesser-Known Facts:**
- GitHub was founded in 2008 and is now owned by Microsoft
- The `.github.io` domain for GitHub Pages was originally created to provide free static website hosting
- GitHub's username format restrictions help maintain clean URLs across the platform
- You can create an unlimited number of public repositories on a free account
