BOOK TO STUDY GUIDE CONVERSION - INSTRUCTIONS
==============================================

OVERVIEW:
- Converts HTML-format books into comprehensive GitHub Markdown study guides
- TWO VERSIONS: Full (with AI enhancements) and Condensed (fluff removed)
- Uses AI to generate quiz questions, learning objectives, and summaries
- Target audience: Beginners who need to become experts efficiently

REQUIREMENTS:
1. Source book must be in HTML format (saved as "Complete Webpage" from browser)
2. Each chapter should be a separate .htm file with associated _files folder for images
3. Python 3 with required libraries (beautifulsoup4, html2text)

AVAILABLE SCRIPTS:

1. convert_book_enhanced.py - FULL VERSION
   - Keeps 100% of original book content
   - Adds AI-generated study guide elements
   - Best for: Reference, comprehensive learning

2. convert_book_condensed.py - CONDENSED VERSION â­ RECOMMENDED
   - Removes 30-40% fluff content
   - Keeps 100% of technical essentials
   - Adds AI-generated study guide elements
   - Best for: Efficient learning, faster reading
   - REUSABLE: Works for any technical book!

WHAT THE CONDENSED VERSION REMOVES:
- Long introductions and welcomes
- Author anecdotes and personal stories
- Repetitive examples
- Verbose explanations and filler phrases
- Marketing content
- Acknowledgments and "about the author" sections

WHAT THE CONDENSED VERSION KEEPS:
- All technical explanations
- All code examples and commands
- All diagrams, images, and tables
- All warnings, tips, and notes
- All best practices
- All security concepts and procedures

STUDY GUIDE STRUCTURE (Both Versions):
Each chapter includes:
1. Title and introduction
2. Learning Objectives (AI-generated from content)
3. Key Points / Interactive Summary
4. Full/Condensed chapter content
5. Quiz Yourself section (5 questions with answers)
6. Additional Resources
7. Navigation links

HOW TO USE - CONDENSED VERSION (Recommended):

1. Save book as HTML format (File > Save As > Webpage, Complete)

2. Edit convert_book_condensed.py:
   - SOURCE_DIR: path to HTML files
   - TARGET_DIR: where to save study guides
   - CHAPTERS: list of (source_file.htm, output_file.md) pairs
   - technical_terms: keywords for your book's domain

3. Run: python convert_book_condensed.py

4. Review output and adjust filtering if needed

CUSTOMIZING FOR DIFFERENT BOOKS:

Programming Books:
technical_terms = ['function', 'class', 'variable', 'method', 'algorithm']

Cloud/DevOps Books:
technical_terms = ['cloud', 'deployment', 'container', 'orchestration']

Security Books (default):
technical_terms = ['security', 'attack', 'threat', 'vulnerability', 'exploit']

Data Science Books:
technical_terms = ['model', 'dataset', 'training', 'prediction', 'algorithm']

See CONDENSED_CONFIG_GUIDE.md for detailed customization options.

ADVANTAGES OF HTML FORMAT:
- Structured content with semantic tags
- Preserved formatting (bold, italic, code)
- Automatic image extraction
- Tables convert to markdown
- Links preserved
- 90% automation vs 50% with plain text

OUTPUT LOCATION:
- Study guides: [TARGET_DIR]/*.md
- Images: [TARGET_DIR]/images/*.png
- Index: [TARGET_DIR]/README.md

EXPECTED RESULTS (Condensed Version):
- Content Reduction: 30-40%
- Reading Time Saved: 30-40%
- Technical Content Lost: 0%
- Learning Efficiency: Significantly improved

SCRIPT LOCATIONS:
All scripts are in the source book directory:
- convert_book_condensed.py (RECOMMENDED - removes fluff)
- convert_book_enhanced.py (keeps everything)
- CONDENSED_CONFIG_GUIDE.md (customization guide)
