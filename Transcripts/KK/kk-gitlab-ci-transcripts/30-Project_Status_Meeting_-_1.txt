Node.js is an open-source runtime environment that enables developers to execute code outside of web browsers.
With Node.js, the developers can create both front-end and back-end applications using JavaScript.
NodeJS is built on top of Chrome V8 JavaScript engine.
This makes it possible to build back-end applications using the same JavaScript programming language you may be familiar with.
NodeJS can be installed on all major platforms including Windows, macOS, and various Linux distributions.
So within NodeJS, when we install it, npm also gets installed.
So npm stands for Node Package Manager.
It is a package manager for JavaScript and NodeJS applications.
It is used to discover, share, distribute, and manage the various packages, libraries, and dependencies that JavaScript developers use when building web and server-side applications.
So let's explore a sample NodeJS project to learn how to run the test and execute NodeJS applications.
Mind you, this is a very basic minimal NodeJS application that displays a simple hello world.
So we have a package.json file in a NodeJS project which contains the metadata including the essential information about the project like its name, version, dependencies, and the scripts.
It is used for dependency management, enabling you to specify which external packages your project relies on developing and deploying your application.
To install the dependencies defined within the package.json, we run the npm install command.
Once the install is successful, a node_modules directory gets created which contains all the external JavaScript modules and packages that your project depends on.
Finally, we have the index.js file where the actual business logic exists and a test.js file is used to define the test cases.
So we have an npm test command which is commonly used to execute all the test cases for a JavaScript project.
This command runs all the tests defined within the test.js file.
Finally, in a NodeJS project, we generally use the npm start command to start your application.
And once the application is started, we can access it on the browser by making a call on the port number.
