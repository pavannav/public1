So what are GitLab Merge Requests? A merge request lets you propose
merging changes from a source branch to a target branch. Does this remind you of anything?
Both GitLab Merge Requests and GitHub Pull Requests serve the same purpose.
They propose changes from a branch to be merged into the main branch of a code repository.
However, they differ in terminology and some workflow implementation details.
But both of these concepts allow the developers to create branching points for the changes,
commit proposed codes like code documentation, etc., for review, discuss changes with the team
and receive the feedback or approvals in some cases. And finally, the changes get reviewed and
potentially merged back into the main codebase. So in the upcoming sessions, we are going to make
use of this merge request concept in GitLab for merging the changes from one branch to another
branch. Thank you.
