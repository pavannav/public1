# 3.16 NAT Gateway High Availability

Hello and welcome to this lecture.So in earlier lecture,we saw how to set up NAT gateway.In this lecture, we are going to seehow to have the highly available NAT gateways.So as you know, NAT gateway scope is AZ levelbecause we are going to create that in a given subnet.That means if one AZ goes down,probably your NAT gateway will also go downand you might lose outbound internet connectivityfor all your instances across multiple availability zones.So, NAT gateways are resilient within a single AZand if you want high availability for NAT gateways,you should create those in multiple AZs.So the architecture will look something like this.This is a VPC and then you have a multiple AZs.In these AZs you have multiple subnets.So NAT gateway resides inside public subnets.So that means you can route trafficfor all EC2 instances in this subnet,private subnet through NAT gateway in the same AZ,and if there is other set of EC2 instances in another AZ,you can route traffic to another NAT in that AZ.So, as I said earlier,very well, these EC2 instances can be routed.There can route a traffic for internetthrough this NAT gateway as well.And this is a legitimate architecture.There is no problem with that,but in case if this AZ goes down,that means these instances,even if they are running,they can't access the internet.So in exam it may be asked that how to havethe high availability for the NAT gateways across AZ.The answer would be to havethe individual NAT gateways in each AZ.So even if AZ goes down,the whole system will go down,and other set of instances and NAT gatewaywill run into another AZ.That means your system is still reliable.So, that is one thing you need to understandabout the NAT gateway high availability for your exam.Later, in the next lecture,we will see how to set up a NAT gatewayon your EC2 instances.Thank you.
