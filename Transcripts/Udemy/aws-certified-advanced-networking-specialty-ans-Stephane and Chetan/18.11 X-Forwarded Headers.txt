# 18.11 X-Forwarded Headers

Okay, so now let's talkabout the X-Forwarded Headersfor HTTP requests on your Load Balancers.So if we take an example of a clientwhich has a public IP,connecting to a Load Balancerwhich has well a public IP to be accessible,but also a private IP on the VPC,then the Load Balancer will connect to EC2 instance.The question is:what is the IP that the EC2 instancewe'll see the requests originating from?Well, because the ALB has a termination mechanism in place,the EC2 instance we'll see traffic originatingfrom the IP that is 10.0.0.25.But if the EC2 instance needs to be ableto identify what is the original IP of the clientsit can do so by looking at some headersthat have been added by the ALB automatically.So, even though the EC2 instance seesthe traffic being from this private address of the ALB,some headers will be edited by the ALB.And it all started with an X-Forwarded prefix.And so we have X-Forwarded-Forwhich is the public IP of the clients.X-Forward-Proto which is a protocol that were usedby the clients to connect into the ALB,and X-Forwarded-Port,which was the port used by the client to connectinto the ALB.So using these headers, you're EC2 instancecan understand where the request came from.So, these are non-standard HTTP headers that are used onlyby the ALB in the CLB.And it's used to enhance the requestand give some information about the clients.You can use these to log client requests on your servers.And this is supported by the CLB in HTTP and HTTPS mode,and the ALB.So, X-Forwarded-For contains the IP address of the clients.And if there are multiple IP addresses,it may have a comma in case you're using proxies.Proto could be HTTP or HTTPS,and a port is a port use to connect to the ELB.So that's it for this lecture, I hope you liked it.And I will see you in the next lecture.
