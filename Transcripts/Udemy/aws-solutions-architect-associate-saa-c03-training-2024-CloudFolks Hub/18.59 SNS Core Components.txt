directly to the multiple system, it send message toan SNS topic, the Publisher only knows the topic, it does not know which multiple systemwill receive this message, the Publisher just says, I am sending this message to topic.Now what happen next is handled by SNS topic.Think of topic, distribution hub, a single entry point for messages, a broadcast channel.Now we have one question, the question is, okay, as per our normal understanding, Publisheris going to send message to topic, but now how topic can send messages to all the systems,we call it subscriber, right.So we have like topic know which system should receive the message?Those system must be registers themselves with the topic.So what we are doing over here, first of all, we will create topic, then all this systemwho is willing to get messages will subscribe to this topic.Once they will become subscriber, they are going to get message from the topic, okay.So system means subscribers going to topic, subscribing itself and then they are goingto get message, okay.So now flow becomes Publisher, Publisher is going to send message to topic and topic isgoing to send message to multiple system, all these systems are actually subscribersof this particular topic.Now let's talk about subscribers.A subscriber is a system that register itself to receive message from the SNS topic.It does not receive message automatically, first it subscribe to the topic, the systemreceive message from the topic only if they subscribe to it.What type of system that can be subscribed to the SNS topic?Now here we have two types of system, right?One AWS service can be subscriber or external system can be subscriber.
