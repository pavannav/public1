# 16.7 How DB Executes Subquery

So now let's understand how the database server executes the subqueries behind the scenes.Let's go.So now let's say that you are a data analyst and you are writing a query at the client side where youhave a subquery inside the main query.So once you go and execute it, what's going to happen.The database engine is going to go and identify the subquery.And in this situation the database is going to execute first the subquery.So here the subquery is like selecting and retrieving data from the table orders.So that means the database has to retrieve the data from the disk storage from the user data.So now once the subquery is executed the result the intermediate result is going to be stored in thecache.So this means the result of the subquery is temporary and as well very fast to retrieve.And now once the database engine is done with the subquery it's going to go and start executing themain query.So let's see in this scenario it's completely depending on the result of the subquery.So that means the main query is going to go and interact with the cache storage.So this means now the data can be retrieved very fast from the result of the subquery.Once it's done it's going to forward the result to the database engine.And the database engine can forward the results to the client side.And at your side you will find the final result.And of course, once everything is executed, the database engine is going to go and clean up the cacheso the subquery results can be destroyed and removed completely from the cache in order to have a freespace for other queries.So this is how the database server executes the subqueries behind the scenes.
