## 24.1 Exam Walkthrough and Signup

### Overview
This module covers the process of signing into your AWS Certification account, creating an AWS Builder ID, navigating the certification portal, and scheduling an AWS certification exam. It explains the options for taking exams in-person at test centers or online via OnVUE, including requirements, preparation steps, and account management through the Pearson VUE platform.

### Key Concepts/Deep Dive

#### Signing Into AWS Certification Account
- Navigate to `aws.training/certification` and click "Sign In"
- Create an AWS Builder ID specifically for AWS certifications by clicking through the process
- After account creation, click "Go to Your Account" to access the certification portal

#### Certification Portal Navigation
- Available options include exam registration, account management, and benefits tracking
- Strongly recommended to watch additional videos on exam accommodations and certification benefits before scheduling

#### Exam Registration Process
- Click "Exam Registration" and select "Schedule an Exam"
- View available exams at the bottom; some may require authorization before scheduling
- Exams appear as "Authorized" or "Click to Authorize" - select the latter and authorize to enable scheduling
- Choose the desired exam (e.g., AWS Certified Developer Associate)

#### Pearson VUE Portal Options
- **In-Person Exam at Test Center**:
  - Physically attend a designated testing center
  - Required to bring photo ID
  - Understand what can and cannot be brought during the exam
  - Click "Next" to proceed
  - Select exam language
  - Agree to conditions and find a testing location by entering an address

- **Online Exam with OnVUE**:
  - Take the exam from home for convenience
  - Requirements: Computer with webcam, good internet connection
  - **Critical**: Run system test well before exam date to ensure compatibility
  - Testing space must be clear of all items (show desk during check-in)
  - Bring photo ID for verification
  - Check in approximately 30 minutes before appointment
  - Select exam language
  - Agree to home exam conditions and remote monitoring
  - Choose language for proctor communication (different from exam language)
  - Specify time zone
  - Select date and time
  - Complete booking and payment
  - Apply any discount vouchers (covered in next module)

#### Exam Launch and Management
- After scheduling, exam appears in "Upcoming Exams"
- Go to Dashboard > "Manage my Pearson VUE account" to access Pearson VUE homepage
- Click "Launch Exam" button when ready to begin

## 24.2 Save 50% on your AWS Exam Cost

### Overview
This module explains how to obtain and apply discount tokens for 50% off future AWS exams after successfully passing an initial certification exam. It details the process of claiming benefits, obtaining voucher codes, and applying them during checkout to reduce exam costs significantly.

### Key Concepts/Deep Dive

#### Certification Benefits System
- Upon successful completion of any AWS exam, tokens are automatically generated
- Tokens provide 50% discount on subsequent AWS exams
- Unclaimed tokens remain available until expiration date
- Benefits become claimable immediately after passing

#### Claiming Discount Benefits
- Navigate to certification portal benefits section
- Identify unclaimed tokens (shown as "Unclaimed")
- Click "Claim Benefits" to activate discount
- After claiming, status changes to "Claimed"
- Each claimed benefit provides a unique voucher code

#### Applying the Discount
- During exam scheduling in Pearson VUE portal, proceed to checkout
- In payment section, locate "Add voucher / promo code" field
- Enter the voucher code obtained from claimed benefits
- Code redemption instantly applies 50% discount to total cost
- Verify discount appears on final total before completing purchase

> [!NOTE]
> This benefit helps reduce certification costs and encourages continuous learning across multiple AWS specialties.

## 24.3 Get an Extra 30 Minutes on your AWS Exam - Non Native English Speakers only

### Overview
Designed specifically for non-native English speakers, this module outlines the process for requesting exam accommodations to receive an additional 30 minutes on AWS certification exams. It covers eligibility requirements, accommodation request procedures, and how to reschedule existing exams to apply the accommodation.

### Key Concepts/Deep Dive

#### Eligibility and Access
- Available for non-native English speakers seeking additional time
- Found under "Exam Registration" > "Exam Accommodations" in certification portal

#### Requesting Accommodation
- Select "ESL +30 minutes" accommodation type
- Submit request through the portal
- Optionally provide documentation to support the accommodation request
- Request is processed and approved if eligible
- Approved accommodation does not expire

#### Applying Accommodation
- Existing exam schedules must be canceled if already booked
- Return to "Schedule Exam" after approval
- Rebook exam - the system automatically accounts for additional 30 minutes
- Total exam duration includes the accommodation time

#### Alternative Accommodations
- For other accommodation needs beyond ESL timing
- Contact Pearson VUE directly through their website
- Request custom accommodations separately from AWS certification portal

> [!IMPORTANT]
> Non-native English speakers should plan to request this accommodation early, as rescheduling may be required and exam slots can fill quickly.

## 24.4 AWS Certification Paths

### Overview
This comprehensive module explores detailed AWS certification pathways tailored to various career roles and professions. It covers foundational, associate, professional, and specialty certifications, providing recommended learning sequences for different job functions including architecture, operations, security, development, networking, data analytics, and AI/ML specialties.

### Key Concepts/Deep Dive

#### Certification Levels Overview
- **Foundational Level**: Entry-point certifications (Cloud Practitioner, AI Practitioner)
- **Associate Level**: Core technical skills (Developer, Solutions Architect, etc.)
- **Professional Level**: Advanced expertise (Solutions Architect Professional, DevOps Engineer Professional)
- **Specialty Level**: Domain-specific deep dives (Security, Advanced Networking, Machine Learning)

#### Career-Specific Certification Paths

##### Solution Architecture Path
**Role**: Design, develop, and manage cloud infrastructure and assets
- Foundational: Cloud Practitioner (optional if cloud/IT experienced)
- AI Integration: AI Practitioner Foundational
- Core: Solutions Architect Associate
- Advanced: Solutions Architect Professional
- Deep Dive: Security Specialty

##### Application Architecture Path
**Role**: Focus on UI, middleware, and infrastructure
- Same foundational as Solution Architecture
- Add: Developer Associate
- Advanced: DevOps Engineer Professional
- Deep Dive: Solutions Architect Professional

##### Systems Administration Path
**Role**: Install, upgrade, maintain computer components and software; implement automation
- Foundational: Cloud Practitioner
- Core: SysOps Administrator Associate
- Advanced: DevOps Engineer Professional

##### Cloud Engineer Path
**Role**: Implement and operate network computing infrastructure; implement security systems
- Foundational: Cloud Practitioner
- Core: SysOps Administrator Associate
- Security: Security Specialty
- Advanced: DevOps Engineer Professional, Advanced Networking Specialty

##### DevOps Engineer Path (Testing Focus)
**Role**: Embed testing and quality practices in software development lifecycle
- Foundational: Cloud Practitioner
- Development: Developer Associate
- Advanced: DevOps Engineer Professional

##### Cloud DevOps Engineer Path
**Role**: Design, deploy, operate large-scale global hybrid cloud environments; advocate CI/CD
- Foundational: Cloud Practitioner
- Core: Developer Associate
- Optional: SysOps Administrator Associate
- AI/ML Integration: Machine Learning Engineer Associate
- Advanced: DevOps Engineer Professional

##### DevSecOps Engineer Path
**Role**: Accelerate cloud adoption with CI/CD principles and security practices
- Foundational: Cloud Practitioner
- Core: SysOps Administrator Associate
- AI/ML: Machine Learning Associate (for AI/ML projects)
- Advanced: DevOps Engineer Professional, Security Specialty

##### Cloud Security Engineer Path
**Role**: Design security architecture and develop cybersecurity solutions
- Foundational: Cloud Practitioner, AI Practitioner Foundational (AI/ML systems)
- Core: SysOps Administrator Associate
- Advanced: Security Specialty
- Deep Dive: DevOps Engineer Professional, Advanced Networking Specialty

##### Cloud Security Architect Path
**Role**: Design enterprise cloud solutions with governance and risk management
- Foundational: Cloud Practitioner, AI Practitioner Foundational
- Core: Solutions Architect Associate
- Advanced: Security Specialty
- Deep Dive: Solutions Architect Professional

##### Software Development Engineer Path
**Role**: Develop and maintain software across platforms and devices
- Foundational: Cloud Practitioner, AI Practitioner Foundational
- Core: Developer Associate
- Advanced: DevOps Engineer Professional

##### Network Engineer Path
**Role**: Design LANs and WANs
- Foundational: Cloud Practitioner
- Core: Solutions Architect Associate
- Specialty: Advanced Networking Specialty
- Deep Dive: Security Specialty

##### Cloud Data Engineer Path
**Role**: Automate collection and processing of structured/semi-structured data; monitor pipelines
- Foundational: Cloud Practitioner
- Core: Solutions Architect Associate
- Data Focus: Data Engineer Associate
- AI/ML Integration: Machine Learning Engineer Associate
- Advanced: Security Specialty

##### Machine Learning Engineer Path
**Role**: Research, build, design AI systems with predictive models
- Foundational: Cloud Practitioner, AI Practitioner Foundational
- Core: Solutions Architect Associate
- ML Focus: Machine Learning Engineer Associate
- Advanced: Data Engineer Associate, Machine Learning Specialty

##### Prompt Engineer Path
**Role**: Design, test, refine text prompts for AI language models
- Foundational: Cloud Practitioner
- Required: AI Practitioner Foundational
- Core: Machine Learning Engineer Associate
- Advanced: Machine Learning Specialty

##### Machine Learning Ops Engineer Path
**Role**: Maintain ML platform and infrastructure
- Foundational: Cloud Practitioner, AI Practitioner Foundational
- Core: Solutions Architect Associate, Machine Learning Associate
- Advanced: Data Engineer Associate, DevOps Engineer Professional

##### Data Scientist Path
**Role**: Develop and maintain AI/ML models for business problems; train and fine-tune models
- Foundational: Cloud Practitioner, AI Practitioner Foundational
- Core: Solutions Architect Associate, Machine Learning Engineer Associate
- Advanced: Machine Learning Specialty

- Contains visual maps and detailed requirements for each career path
- Recommended to review based on specific role objectives
